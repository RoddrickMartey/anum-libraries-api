Controller Error Handling Structure
===================================

Location:
- All branch and network controllers live under `src/branch/controllers` and `src/network/controllers`.

Common pattern:
1. Validate input where needed using Zod schemas.
   - `safeParse(req.body)`
   - return `422` with `VALIDATION_ERROR` and `details` if invalid.

2. Use `try { ... } catch (error) { ... }` around service calls.
   - Most controller functions are `async` and return `Promise<void>`.
   - The try block includes authorization checks and service invocation.

3. Early authorization checks return `403`.
   - Typical guard: `if (!branchId)` or missing `createdBy`.
   - Response shape: `{ error: 'Forbidden', code: 'FORBIDDEN' }`.

4. Handle known domain errors by inspecting `error.message`.
   - Example: `if (error instanceof Error && error.message === 'BOOK_NOT_FOUND') { ... }`
   - Common mapped statuses:
     - `404` resource not found
     - `409` conflict / resource already exists
     - `401` invalid credentials or unauthorized action

5. Fallback on server error logging and `500`.
   - `logger.error(...)` is called with the caught error.
   - Response shape: `{ error: 'Internal server error', code: 'INTERNAL_SERVER_ERROR' }`.

6. Success responses are returned with `res.status(...).json(...)` or `res.status(204).send()`.
   - Normal success types: `200`, `201`, `204`.
   - JSON envelope usually includes `data`, `pagination`, or `message`.

Specific details from the current codebase:
- `books.controller.ts` uses `BOOK_NOT_FOUND` for missing book resources.
- `staff.controller.ts` handles `STAFF_NOT_FOUND`, `EMAIL_ALREADY_EXISTS`, and `INVALID_CURRENT_PASSWORD`.
- `network/controllers/branches.controller.ts` handles `BRANCH_NOT_FOUND` and `ADMIN_EMAIL_TAKEN`.

Overall design:
- Error handling is controller-local rather than centralized middleware-based.
- The controllers return structured JSON error objects consistently.
- Domain-specific errors are communicated through error message checks and dedicated error codes.
